Git Workflow עם Claude Code

Claude Code מבין Git באופן מלא — הוא יכול ליצור ברנצ'ים, לכתוב commit messages, לפתוח PRs, ולהגדיר hooks שישמרו על איכות הקוד בכל קומיט.

⏱ 10 דקות קריאה רמה: בינוני

עבודה עם ברנצ'ים

הדרך הנכונה לעבוד עם Claude Code היא תמיד על ברנץ' נפרד — כך קל לבדוק את השינויים לפני מיזוג ל-main.

שיחה עם claude 
create a new branch called feat/user-auth and implement
JWT authentication middleware for the Express app

Claude ייצור את הברנץ', יכתוב את הקוד, ויבצע קומיט עם הודעה תיאורית. ניתן גם לבקש ממנו לחלק את העבודה למספר קומיטים לוגיים.

טיפ בקשו מ-Claude לעבוד בקומיטים קטנים עם הודעות ברורות. זה עוזר לעשות code review ולחזור לנקודות מסוימות אם צריך.

Commit Messages אוטומטיים

Claude כותב commit messages לפי Conventional Commits כברירת מחדל, אבל ניתן להתאים את הסגנון ב-CLAUDE.md:

markdown — CLAUDE.md 
## Git Conventions
- Commit format: `type(scope): description`
- Types: feat, fix, docs, refactor, test, chore
- Max subject line: 72 chars
- Always write in English
- Include issue number if relevant: `fix(auth): resolve token expiry (#42)`

יצירת PR אוטומטית

עם GitHub MCP מותקן, Claude יכול לפתוח PR עם תיאור מלא בלחיצה אחת:

  1. סיום הפיצ'ר

    לאחר שהקוד מוכן ונבדק, מבקשים מ-Claude לסכם את השינויים:

    summarize all changes made in this branch compared to main
  2. פתיחת PR

    Claude ישתמש ב-GitHub MCP ליצור PR עם כותרת ותיאור:

    open a PR from feat/user-auth to main.
mark it as draft if tests haven't passed yet
  3. Code Review אוטומטי

    ניתן לבקש מ-Claude לעבור על ה-diff לפני שליחה:

    review the diff of this branch. flag any security issues,
missing error handling, or tests that should be added

Git Hooks לאבטחת קומיטים

Git hooks הם סקריפטים שרצים אוטומטית לפני או אחרי פעולות Git. Claude יכול ליצור אותם כדי לאכוף איכות קוד.

שיחה עם claude 
create a pre-commit hook that:
1. runs eslint and blocks commit if there are errors
2. runs prettier and auto-formats changed files
3. checks for console.log statements and warns (but doesn't block)

Claude ייצור את הקובץ .git/hooks/pre-commit עם כל הלוגיקה:

bash — .git/hooks/pre-commit 
#!/bin/sh
set -e

# Run ESLint on staged files
STAGED=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(js|ts|jsx|tsx)$')
if [ -n "$STAGED" ]; then
  echo "$STAGED" | xargs npx eslint --max-warnings 0
fi

# Auto-format with Prettier
if [ -n "$STAGED" ]; then
  echo "$STAGED" | xargs npx prettier --write
  echo "$STAGED" | xargs git add
fi
שימו לב אחרי יצירת ה-hook, יש להפוך אותו לקובץ הניתן להרצה: chmod +x .git/hooks/pre-commit. Claude יזכיר לכם זאת אוטומטית.

Rebase ו-Conflict Resolution

Claude יכול לעזור גם במקרים מסובכים — rebase, merge conflicts, וניקוי היסטוריה:

שיחה עם claude 
I have merge conflicts in src/api/users.ts and src/models/user.ts.
resolve them — keep the incoming changes for the schema fields
but keep my version of the validation logic
