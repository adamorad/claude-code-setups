בדיקות Playwright עם Claude Code

Playwright הוא הכלי המוביל לאוטומציה של דפדפן ובדיקות E2E. Claude יכול לכתוב בדיקות, לנפות שגיאות, ולצלם screenshots — עם MCP שמחבר אותו ישירות לדפדפן.

⏱ 11 דקות קריאה רמה: בינוני

התקנת Playwright

התקנה מהירה של Playwright עם כל הדפדפנים הנדרשים:

bash 
npm init playwright@latest
# בחרו: TypeScript, tests/ directory, GitHub Actions CI

לאחר מכן, מוסיפים את Playwright MCP כדי ש-Claude יוכל לשלוט בדפדפן ישירות:

json — claude_desktop_config.json 
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}

כתיבת בדיקות E2E

Claude יכתוב בדיקות E2E מתיאור פשוט בעברית. הוא מכיר את ה-best practices של Playwright — שימוש ב-role locators, המתנה לאלמנטים, וטיפול ב-async.

שיחה עם claude 
write E2E tests for the user registration flow:
1. navigate to /register
2. fill email, password, confirm password
3. submit and verify redirect to /dashboard
4. check that welcome email toast appears
test both happy path and error cases (duplicate email, weak password)
typescript — תוצאה 
import { test, expect } from '@playwright/test';

test.describe('User Registration', () => {
  test('successful registration redirects to dashboard', async ({ page }) => {
    await page.goto('/register');

    await page.getByRole('textbox', { name: 'Email' }).fill('new@example.com');
    await page.getByRole('textbox', { name: 'Password' }).fill('SecurePass123!');
    await page.getByRole('textbox', { name: 'Confirm' }).fill('SecurePass123!');
    await page.getByRole('button', { name: 'Create account' }).click();

    await expect(page).toHaveURL('/dashboard');
    await expect(page.getByRole('alert')).toContainText('ברוכים הבאים');
  });

  test('duplicate email shows error', async ({ page }) => {
    await page.goto('/register');
    await page.getByRole('textbox', { name: 'Email' }).fill('existing@example.com');
    await page.getByRole('textbox', { name: 'Password' }).fill('SecurePass123!');
    await page.getByRole('button', { name: 'Create account' }).click();

    await expect(page.getByRole('alert')).toContainText('כתובת המייל כבר קיימת');
  });
});

רגרסיה ויזואלית

Playwright תומך בהשוואה ויזואלית — בדיקה שהממשק לא השתנה בין גרסאות. Claude יגדיר את הבדיקות ואת ה-threshold:

  1. יצירת Snapshots בסיסיים 
    add visual regression tests for:
- homepage hero section
- pricing page (all 3 tiers visible)
- dashboard main view (logged in state)
set threshold to 0.1% pixel difference
  2. הרצת בדיקות ויזואליות 
    npx playwright test --update-snapshots  # יצירת baseline
npx playwright test                      # בדיקה מול baseline
  3. ניתוח כשלונות

    כאשר בדיקה ויזואלית נכשלת, שולחים ל-Claude את ה-diff:

    the visual test for homepage failed. here's the diff image path:
test-results/homepage-diff.png
what changed and is it expected?

שימוש ב-MCP לדיבוג חי

עם Playwright MCP, Claude יכול לנווט לאתר שלכם ולבדוק בעיות בזמן אמת — בלי לכתוב קוד:

שיחה עם claude 
go to http://localhost:3000/checkout and complete a purchase
using test card 4242 4242 4242 4242. take a screenshot at each step
and tell me if anything looks broken
טיפ — CI Integration הוסיפו את הבדיקות ל-GitHub Actions כדי שיריצו בכל PR. Claude יכתוב את קובץ ה-workflow עבורכם — בקשו: "add Playwright tests to GitHub Actions CI".
ביצועים בדיקות E2E הן איטיות — הריצו אותן במקביל עם workers: 4 ב-playwright.config.ts. בקשו מ-Claude לאופטמז את ה-config.
