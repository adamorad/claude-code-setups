בניית API עם Claude Code

עם Context7 MCP, Claude מקבל גישה לתיעוד עדכני של FastAPI, Express, ו-frameworks אחרים — ולא מסתמך על ידע ישן. התוצאה: קוד שעובד עם הגרסה שלכם, לא עם גרסה ישנה.

הגדרת Context7 MCP

Context7 הוא MCP שמביא תיעוד עדכני של ספריות ישירות לתוך ההקשר של Claude — כך הוא יכול לכתוב קוד שמתאים לגרסה שאתם משתמשים בה.

json — claude_desktop_config.json 
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    }
  }
}

לאחר ההגדרה, מציינים בשיחה שרוצים שימוש ב-Context7:

use context7 to get the latest FastAPI docs, then build a REST API

API עם FastAPI (Python)

FastAPI הוא framework פייתון מודרני עם תיעוד אוטומטי (Swagger/OpenAPI). Claude יבנה את כל המבנה:

  1. מבנה הפרויקט 
    create a FastAPI project structure for a task management API:
- routers/ (tasks, users, auth)
- models/ (SQLAlchemy)
- schemas/ (Pydantic)
- dependencies.py (DB session, current user)
use context7 for the latest FastAPI patterns
  2. endpoint ראשון עם validation 
    # routers/tasks.py
from fastapi import APIRouter, Depends, HTTPException, status
from sqlalchemy.orm import Session
from ..dependencies import get_db, get_current_user
from ..schemas import TaskCreate, TaskResponse
from ..models import Task, User

router = APIRouter(prefix="/tasks", tags=["tasks"])

@router.post("/", response_model=TaskResponse, status_code=201)
async def create_task(
    task: TaskCreate,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    db_task = Task(**task.model_dump(), owner_id=current_user.id)
    db.add(db_task)
    db.commit()
    db.refresh(db_task)
    return db_task
  3. תיעוד אוטומטי

    FastAPI מייצר Swagger UI אוטומטית. Claude יוסיף תיאורים ודוגמאות: 

    add docstrings and example values to all endpoints so the
auto-generated Swagger docs are useful for API consumers

API עם Express (Node.js)

לפרויקטים ב-TypeScript/Node.js, Claude יעבוד עם Express ו-Zod לvalidation:

שיחה עם claude 
build an Express REST API for a blog platform:
- POST /posts (create, auth required)
- GET /posts (list, with pagination and filters)
- GET /posts/:slug (single post)
- PUT /posts/:id (update, owner only)
- DELETE /posts/:id (soft delete)
use Zod for request validation, return consistent error shapes
typescript — תוצאה חלקית 
// routes/posts.ts
import { Router } from 'express';
import { z } from 'zod';
import { validate } from '../middleware/validate';
import { requireAuth } from '../middleware/auth';
import { PostService } from '../services/PostService';

const router = Router();

const CreatePostSchema = z.object({
  title: z.string().min(3).max(200),
  content: z.string().min(10),
  tags: z.array(z.string()).max(5).optional(),
  published: z.boolean().default(false),
});

router.post('/', requireAuth, validate(CreatePostSchema), async (req, res) => {
  const post = await PostService.create({
    ...req.body,
    authorId: req.user.id,
  });
  res.status(201).json({ data: post });
});

export default router;

בדיקות API אוטומטיות

Claude יכתוב בדיקות לכל endpoint — unit tests ו-integration tests:

שיחה עם claude 
write pytest tests for all task endpoints.
use httpx for async requests, use fixtures for test DB and auth tokens.
aim for 90% coverage of the routers/ directory
טיפ — OpenAPI ללא קוד אפשר לבקש מ-Claude ליצור OpenAPI spec מתיאור טקסטואלי בלבד, ואז לייצר ממנו קוד שרת ו-client SDK אוטומטית עם openapi-generator.

Rate Limiting ו-Auth

כל API בייצור צריך rate limiting ו-authentication. Claude יוסיף אותם בצורה נכונה:

שיחה עם claude 
add to the Express API:
1. JWT auth middleware (verify Bearer token, attach user to req)
2. rate limiting: 100 req/min per IP globally, 20 req/min per user for write endpoints
3. request logging with response time
4. helmet for security headers
לפני Production בקשו מ-Claude לעשות security review לפני העלאה: "review all API endpoints for common vulnerabilities: SQL injection, missing auth checks, sensitive data exposure, and missing input sanitization".
