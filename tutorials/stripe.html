Payments

תשלומי Stripe עם Claude Code

Stripe MCP מאפשר ל-Claude לבדוק תשלומים, לנהל customers, ולפתור בעיות — ישירות מהשיחה. בנוסף, Claude יכול לכתוב את כל קוד האינטגרציה עם Stripe SDK.

⏱ 12 דקות קריאה רמה: בינוני

הגדרת Stripe MCP

Stripe מספקת MCP רשמי שמאפשר גישה ל-API שלהם ישירות מ-Claude:

json — claude_desktop_config.json 
{
  "mcpServers": {
    "stripe": {
      "command": "npx",
      "args": ["-y", "@stripe/agent-toolkit/mcp"],
      "env": {
        "STRIPE_SECRET_KEY": "sk_test_..."
      }
    }
  }
}
השתמשו תמיד ב-Test Mode המפתח צריך להתחיל ב-sk_test_ ולא ב-sk_live_. לעולם אל תתנו ל-MCP גישה ל-live keys בסביבת פיתוח. לכו ל-Stripe Dashboard להשגת test key.

בניית מנויים (Subscriptions)

Claude יכול לכתוב את כל הקוד הנדרש למנויים — יצירת products, prices, וחיבור Checkout:

  1. יצירת Products ו-Prices 
    create a Stripe subscription setup with 3 tiers:
- Free: $0/month, 100 API calls
- Pro: $29/month, 10,000 API calls
- Enterprise: $99/month, unlimited
create the products and prices in test mode via the MCP
  2. Checkout Session

    Claude יכתוב את הקוד לפתיחת Checkout:

    // server/stripe/checkout.ts
export async function createCheckoutSession(
  customerId: string,
  priceId: string,
  successUrl: string,
  cancelUrl: string
) {
  const session = await stripe.checkout.sessions.create({
    customer: customerId,
    mode: 'subscription',
    line_items: [{ price: priceId, quantity: 1 }],
    success_url: successUrl,
    cancel_url: cancelUrl,
    subscription_data: {
      trial_period_days: 14,
    },
  });
  return session.url;
}
  3. Customer Portal

    לאפשר למשתמשים לנהל את המנוי שלהם:

    add a "manage subscription" button that redirects to
the Stripe Customer Portal for self-service cancellation and upgrades

Webhooks

Webhooks הם האופן שבו Stripe מודיע לאפליקציה שלכם על אירועים — תשלום הצליח, מנוי בוטל, חיוב נכשל. Claude יכתוב את כל הלוגיקה:

שיחה עם claude 
write a webhook handler for these Stripe events:
- payment_intent.succeeded: activate user subscription in DB
- customer.subscription.deleted: downgrade user to free tier
- invoice.payment_failed: send email and mark account as past_due
include signature verification
typescript — תוצאה 
// server/webhooks/stripe.ts
export async function handleStripeWebhook(req: Request, res: Response) {
  const sig = req.headers['stripe-signature']!;
  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(
      req.body,
      sig,
      process.env.STRIPE_WEBHOOK_SECRET!
    );
  } catch (err) {
    return res.status(400).send(`Webhook Error: ${err.message}`);
  }

  switch (event.type) {
    case 'payment_intent.succeeded':
      await activateSubscription(event.data.object as Stripe.PaymentIntent);
      break;
    case 'customer.subscription.deleted':
      await downgradeToFree(event.data.object as Stripe.Subscription);
      break;
    case 'invoice.payment_failed':
      await handleFailedPayment(event.data.object as Stripe.Invoice);
      break;
  }

  res.json({ received: true });
}

פתרון בעיות עם MCP

כאשר יש בעיה בתשלום, Claude יכול לחקור ישירות דרך ה-MCP:

שיחה עם claude 
customer cus_ABC123 says their subscription is active but
they can't access pro features. check their subscription status,
last payment, and any failed invoices in Stripe
טיפ — בדיקת Webhooks מקומית השתמשו ב-Stripe CLI לבדיקת webhooks בסביבה מקומית: stripe listen --forward-to localhost:3000/webhooks/stripe. בקשו מ-Claude לעזור בהגדרת הכלי.
