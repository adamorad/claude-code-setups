התחלה עם Claude Code

Claude Code הוא CLI שמאפשר לכם לעבוד עם Claude ישירות מהטרמינל — כותב קוד, מנהל קבצים, מריץ פקודות, ומבין את כל הפרויקט בהקשר אחד.

⏱ 8 דקות קריאה רמה: מתחיל

התקנה

Claude Code מחייב Node.js 18 ומעלה. ניתן להתקין דרך npm:

bash 
npm install -g @anthropic-ai/claude-code

לאחר ההתקנה, מריצים claude בטרמינל ומתחברים עם API key של Anthropic. ניתן לקבל מפתח ב-console.anthropic.com.

bash 
claude
# או עם API key ישירות:
ANTHROPIC_API_KEY=sk-ant-... claude
טיפ מוסיפים את ANTHROPIC_API_KEY ל-~/.zshrc או ~/.bashrc כדי לא להקליד בכל פעם.

קובץ CLAUDE.md

CLAUDE.md הוא קובץ הגדרות שנמצא בתיקיית הפרויקט (או ב-~/.claude/ להגדרות גלובליות). Claude קורא אותו אוטומטית בכל שיחה ומשתמש בו כהקשר.

זה המקום להגדיר: שפת הפרויקט, conventions, פקודות בנייה, כללי סגנון קוד, ומה לא לגעת בו.

markdown — CLAUDE.md 
# My Project

## Stack
- Python 3.12, FastAPI, PostgreSQL
- Tests: pytest
- Linting: ruff

## Commands
- Run: `uvicorn main:app --reload`
- Test: `pytest -v`
- Lint: `ruff check .`

## Style
- Type hints everywhere
- No print() — use logging
- Max line length: 100
טיפ ניתן ליצור CLAUDE.md אוטומטית עם הפקודה claude init בתיקיית הפרויקט.

פרויקט ראשון

הדרך הטובה ביותר להתחיל היא לפתוח טרמינל בתיקיית פרויקט קיים ולהריץ claude. Claude יסרוק את הקבצים ויהיה מוכן לשאלות.

  1. פתיחת Claude בפרויקט

    ניגשים לתיקיית הפרויקט ומפעילים את ה-CLI:

    cd ~/my-project
claude
  2. שאלה ראשונה

    מתחילים עם שאלה כללית על הפרויקט כדי לוודא ש-Claude הבין את המבנה: 

    explain the project structure and main entry points
  3. משימה אמיתית

    עוברים למשימה ספציפית. Claude יכתוב קוד, יריץ בדיקות, ויתקן שגיאות אוטומטית: 

    add a /health endpoint that returns {"status": "ok", "version": "1.0"}
then run the tests to make sure nothing broke
  4. מצב Headless לאוטומציה

    ניתן להריץ Claude ללא אינטראקציה — מושלם לסקריפטים ו-CI:

    claude --print "run tests and return exit code"

קיצורי דרך שימושיים

בתוך שיחה עם Claude Code יש מספר פקודות מובנות שמאיצות את העבודה:

claude commands 
/help          — רשימת כל הפקודות
/clear         — ניקוי ההיסטוריה של השיחה
/compact       — דחיסת ההקשר לחסוך tokens
/model         — החלפת מודל (Sonnet / Opus / Haiku)
/vim           — עריכת prompt ב-vim
Ctrl+C         — עצירת פעולה באמצע
שימו לב Claude Code יכול לכתוב ולמחוק קבצים. עדיף לעבוד בתוך git repo כדי שתמיד תוכלו לבצע git diff ו-git checkout אם משהו לא הלך כמו שציפיתם.
